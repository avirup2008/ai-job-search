### 1. Source Quality Panel Visual Rendering

**Test:** Visit `/analytics` in local dev (`npm run dev`) or production after at least one nightly pipeline run has discovered jobs.
**Expected:** "Source quality" section appears in the panel grid below the budget panel. Horizontal bars are visible for each discovery source with a width proportional to T1 count. Each row shows the source display name (e.g. "Adzuna", "Magnet.me"), a filled bar, and the label "{n} T1 ({rate}%)". When no jobs exist, the empty state "No jobs discovered yet." appears instead of an empty bar list.
**Why human:** Next.js Server Component rendering, CSS `hbar-*` layout, and the conditional empty-state branch all require a live browser session against a real (or seeded) database.

### 2. Market Pulse Panel — Data Branches

**Test:** Visit `/analytics`. Check each of three distinct data states: (a) no applications exist — panel shows "No responses yet" for avg days-to-response; (b) applications exist but none past "applied" status — same; (c) at least one application has reached interview/offer/rejected status — shows numeric day count. Also verify the T1 trend label reads "No data yet" when no T1 jobs exist, and that per-source response rate rows render when applications are linked to jobs with a known source.
**Expected:** Each data state surfaces the correct UI branch: null avgDays → "No responses yet"; numeric avgDays → "{n} days"; empty sourceResponseRate → no rows; populated sourceResponseRate → one hbar-row per source.
**Why human:** The conditional rendering at `avgDaysToResponse !== null` and the `.map()` over `sourceResponseRate` only exercise specific branches based on live database state that cannot be mocked in an automated check.

### 3. Weekly Strategy Brief — Cron to Card End-to-End

**Test:** On a Monday, call `GET /api/cron/weekly-brief` with `Authorization: Bearer {CRON_SECRET}`. Then visit `/analytics`.
**Expected:** (a) The cron endpoint returns `{ ok: true, brief: { ... } }` with status 200; a row appears in the `runs` table with `status = 'weekly-brief'` and a populated `stageMetrics` JSON object. (b) The "Weekly strategy brief" panel on `/analytics` transitions from the empty state to showing "Week of {YYYY-MM-DD}", the three data rows (applications/target, T1 available vs applied, top source), and the callout string. On a non-Monday, the endpoint returns `{ ok: true, skipped: "not monday" }` and no row is inserted.
**Why human:** The Monday guard (`getUTCDay() !== 1`) and the runs table read-back path can only be exercised end-to-end by actually calling the endpoint on the correct day or temporarily bypassing the guard in a dev environment.
