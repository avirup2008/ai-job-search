# Quick 260417-o87: Close Codex Review Gap 93→97 — Auth Tests + Ops + Schema

**One-liner:** Auth/session test suite (19 tests), per-source orchestrator metrics + CV error logging, typed storage columns (storageUrl/format/mimeType/renderKind) with migration 0004, and COOKIE_NAME/computeSessionToken constants extraction eliminating all raw "disha_session" literals.

## What Was Built

### Task 1 — Auth/session test suite (19 tests)
- `tests/unit/auth/middleware.test.ts` — 10 tests: 6 page redirects (inbox/pipeline/analytics/profile/budget/paste), 3 API 401s (generate/download-pack/paste-role), 1 authenticated pass-through. Uses `vi.spyOn(globalThis.crypto.subtle, "digest")` to stub Web Crypto in Vitest's Node environment.
- `tests/unit/auth/login.test.ts` — 4 tests: correct password → 200 + HttpOnly cookie, wrong password → 401 + no cookie, invalid JSON → 400, logout → redirect + cleared cookie.
- `tests/unit/auth/page-auth.test.ts` — 5 tests: mirror of isAuthenticated() guard logic verifying wrong-length cookie, garbage bytes, missing env, valid token, undefined cookie — all without throwing.

### Task 2 — Operational diagnostics improvements
- `orchestrator.ts`: `RunSummary` extended with `sourceSummary: SourceSummaryEntry[]` (source/fetched/error per source) and `cvErrors?: Array<{jobId, documentKind, error}>`. CV failures now appended to `cvErrors` and written to `errorJson` column.
- `purge/route.ts`: After `purgeOldJobs()`, inserts a `runs` row with `status="purge"` and `stageMetrics={deletedJobs, deletedDocuments, ranAt, dryRun}` so purge outcomes are observable.
- `src/app/api/admin/last-run/route.ts` (new): GET handler returning most recent run's `stageMetrics` + `errorJson`, ordered by `startedAt DESC`. Returns 404 if no runs exist.

### Task 3 — Document/storage contract fix
- `schema.ts`: Added 4 columns to `documents` table — `storageUrl`, `format`, `mimeType`, `renderKind`. Deprecation comment on `blobUrlDocx`.
- `0004_add_storage_url_columns.sql`: `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` for all 4 columns; backfill `storage_url` from `blob_url_docx`; infer `format`/`mime_type`/`render_kind` from `kind` column.
- `storage.ts`: All 5 store functions (`storeArtifact`, `storeCv`, `storeCoverLetter`, `storeScreeningQA`, `storeInterviewPrep`) write `storageUrl`+`format`+`mimeType`+`renderKind`. Stale repurposing comments replaced with backward-compat notes. `deleteInterviewPrep` now prefers `storageUrl ?? blobUrlDocx`.

### Task 4 — Constants extraction and cleanup
- `src/lib/auth/constants.ts` (new): Exports `COOKIE_NAME = "disha_session" as const` and `computeSessionToken(password)` (sha256 hex via node:crypto).
- All 4 auth files (`middleware.ts`, `login/route.ts`, `logout/route.ts`, `page.tsx`) import `COOKIE_NAME` — no raw `"disha_session"` literals remain.
- `login/route.ts` and `page.tsx` use `computeSessionToken()` instead of inline `createHash` calls.
- Edge runtime comment added to middleware explaining why Web Crypto is kept inline.
