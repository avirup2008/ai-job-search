<objective>
Add a 60-day data retention cron to the Disha app that purges jobs whose applications are in terminal/untouched states (rejected, discarded, saved, new) and have been idle >60 days. Cascade-delete associated documents (cover letters, CVs, screening QA, artifacts) from Vercel Blob before the DB rows are removed. Ship behind an env-gated dry-run so the first production run observes-only.

Purpose: Keep Neon DB + Blob storage bounded over time; preserve applied/interviewing/offered pipeline forever.
Output: New `/api/cron/purge` endpoint, retention lib, Vercel Cron entry, tests, all shipped with `RETENTION_DRY_RUN=true` default.
</objective>
